Case Study

PulseDesk — daily KPI digest for the leadership team

PulseDesk is a fast-growing SaaS analytics tool used by 4,200 companies. Their lead analyst, Anaya Iyer, sends the founders a daily email at 9:00 AM with signups, churn, MRR and active users. She also wants instant alerts when churn spikes above 4%. Doing this by hand every morning is a chore — let's automate it.

1

How an email actually gets sent — the SMTP conversation

When Python calls smtplib.SMTP(...), it opens a chat with the mail server. The server and your code exchange short, numbered messages — like a polite handshake. Watch it happen.

Think of SMTP (Simple Mail Transfer Protocol) as a strict receptionist at a hotel. You walk in (EHLO), show ID (AUTH), tell them who's sending and who's receiving (MAIL FROM, RCPT TO), hand over the letter (DATA), and say goodbye (QUIT). Every step has a 3-digit response code: 2xx = OK, 3xx = need more info, 5xx = rejected.

import smtplib from email.message import EmailMessage msg = EmailMessage() msg["From"] = "anaya@pulsedesk.in" msg["To"] = "founders@pulsedesk.in" msg["Subject"] = "Daily KPI digest — 25 May" msg.set_content("Signups: 187 | Churn: 3.1% | MRR: ₹42.6L") with smtplib.SMTP("smtp.gmail.com", 587) as s: s.starttls() # upgrade to encrypted s.login("anaya@pulsedesk.in", APP_PWD) s.send_message(msg) # DATA + QUIT under the hood
Notice how port 587 and starttls() always travel together — that's the modern, encrypted way. Port 465 with SMTP_SSL is the older alternative. Never use port 25 from your laptop; ISPs block it.

Plain text vs HTML — the same email, two looks

A plain-text email is just a string. An HTML email is a tiny webpage that arrives in the inbox. The founders skim KPIs on a phone over breakfast — make those numbers easy to scan with colour, layout, and a clear hierarchy.

Modern email libraries let you attach both versions at the same time. The recipient's email client picks whichever it can render. Outlook, Gmail and Apple Mail almost always pick HTML — but old terminal clients fall back to plain text gracefully.

Email HTML is not normal web HTML — Gmail, Outlook and Apple Mail each strip away different things. Stick to inline styles, simple tables for layout, and avoid <script> tags entirely (they're always blocked).

Attach a chart, a CSV, a PDF report

The founders sometimes want the raw numbers, sometimes a chart, sometimes a polished PDF. Click items from the shelf to attach them — watch the email compose itself.

Every attachment has two pieces of metadata: the MIME type (e.g. application/pdf, image/png) and the filename. The email client uses these to decide which icon to show and which app to open it with. Charts saved with matplotlib.savefig() are just PNG files — they attach the same way as any image.

from email.message import EmailMessage import mimetypes msg = EmailMessage() msg["Subject"] = "Daily KPI digest" msg.set_content("See attached files.") files = ["kpi_report.pdf", "signups_chart.png", "raw_signups.csv"] for path in files: ctype, _ = mimetypes.guess_type(path) # e.g. "image/png" maintype, subtype = ctype.split("/", 1) with open(path, "rb") as f: msg.add_attachment( f.read(), maintype=maintype, subtype=subtype, filename=path, )
Most providers cap attachments at 25 MB. For charts, save matplotlib figures as PNG at 100 DPI — large enough to look crisp, small enough to attach. For bigger files, upload to S3/Drive and paste the link instead.
4

Threshold alerts — fire an email when a number crosses a line

A daily digest is one thing — but if churn suddenly hits 5%, the founders want to know now, not at 9 AM tomorrow. Drag the threshold slider below the current value to trigger an alert.

The pattern is dead simple: a script runs every few minutes, reads the current metric, compares it to the threshold, and only sends an email if the line is crossed. The hard part isn't the code — it's choosing good thresholds (too sensitive = inbox spam, too loose = late warnings) and preventing repeated alerts (don't email every 5 minutes for the same breach).

import schedule, time from notifier import send_email THRESHOLD = 4.0 # churn % limit last_alert_at = 0 # prevent spam: 1 alert / hour def check_churn(): global last_alert_at churn = read_metric_from_db("churn_rate") if churn > THRESHOLD and time.time() - last_alert_at > 3600: send_email( to="founders@pulsedesk.in", subject=f"🚨 Churn alert: {churn}% > {THRESHOLD}%", body=build_alert_html(churn, THRESHOLD), ) last_alert_at = time.time() schedule.every(5).minutes.do(check_churn) while True: schedule.run_pending() time.sleep(30)
A common upgrade: tiered severity. Churn > 4% → email founders. Churn > 6% → email founders + Slack #urgent + SMS to the on-call analyst. Add a "cooldown" of 1 hour so the same breach doesn't fire 12 emails in an hour.
