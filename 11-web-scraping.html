Web Scraping with BeautifulSoup
Module 11 · Scraping

The web is a giant spreadsheet in disguise.

When no friendly API exists, websites themselves become your data source. Web scraping is the art of reading the page like a computer would — finding the rows, picking the columns, and politely walking away. We'll keep it ethical, lean, and Python-light.

🏠
The Case · FlatFinder

Tracking rental listings across four city portals every morning

FlatFinder is a tiny startup helping families in Pune relocate. Each morning, an analyst manually copies 60+ listings from four property sites into a spreadsheet — locality, BHK, rent, and link. It takes two hours. We'll teach a Python script to do it in two minutes, ethically.

1HTML structure · how a webpage is really built

The same page, twice: what you see, and what the browser reads

Behind every webpage is a tree of nested tags. The browser turns that tree into the rectangles, text, and images you see. To scrape, you need to see both at once. Click any item on the left and watch its HTML light up on the right.

Every HTML element is a tag (like <h2> or <span>) with optional attributes (like class="price") and content (the text or other tags inside it). Tags nest inside other tags, forming a tree. Scraping = "find the tags I want, pull out their text".

Use the browser's "Inspect" tool first

Before writing a single line of scraping code, open the actual page in Chrome or Firefox, right-click the data you want, and choose Inspect. The dev tools panel shows the exact HTML tag, class, and ID. That's the recipe your BeautifulSoup code will follow. Five minutes inspecting saves an hour debugging.

2Tag and class selectors · saying "I want these"

Five tiny patterns that cover 90% of scraping

Selectors are sentences like "give me every span with class price". Pick a preset below, then try writing your own. The matches highlight live, exactly as BeautifulSoup would return them.

Five patterns to memorize: tag (h3 matches every h3), class (.price matches every element with class="price"), id (#top-result matches that one element), tag.class (span.price = span elements with class price), and parent > child (.listing h3 = h3 inside .listing). These work in browser dev tools, in BeautifulSoup's .select(), and in CSS itself.
Pick a selector · see what matches in the 3 listings
3 matches
from bs4 import BeautifulSoup

soup = BeautifulSoup(html, "html.parser")

# By tag — every h3 on the page
titles = soup.find_all("h3")
for t in titles: print(t.text.strip())

# By class — every element with class="price"
prices = soup.find_all(class_="price")

# By tag + class — span elements with class="price"
prices = soup.find_all("span", class_="price")

# CSS-style selectors with .select() — most flexible
listings   = soup.select(".listing")         # every listing card
titles     = soup.select(".listing h3")      # h3 inside .listing
first_meta = soup.select_one(".meta span")   # first match only

# Pull out the structured data row by row
rows = []
for card in soup.select(".listing"):
    rows.append({
        "title": card.select_one("h3").text.strip(),
        "price": card.select_one(".price").text.strip(),
    })

The find_one-then-find_all pattern

The cleanest scrapers follow one pattern: first select the outer container (the listing card), then for each card pull out the specific fields with smaller selectors. Don't try to grab all the prices and all the titles separately — if the third listing is missing a price, your two lists go out of sync and every row after that is wrong. Group by container first.

3Extracting tables · the one-line shortcut

Why scraping tabular pages can be embarrassingly easy

When a page already contains a real HTML <table>, you don't need BeautifulSoup at all. Pandas has a one-liner that reads every table on a page directly into a DataFrame. Below is FlatFinder's "neighborhood averages" page — click extract.

Pandas's pd.read_html(url) finds every <table> on a page and returns a list of DataFrames. Index [0] usually gives you the main one. This works for stock screeners, Wikipedia tables, government data dashboards, NSE/BSE pages, RBI reports — anywhere the data is already in an HTML table.

Pandas auto-detected the header row, converted the comma-formatted numbers to integers, and built a clean DataFrame in one line.
import pandas as pd

# ─── Easiest path: pandas reads every table on the page ────
url = "https://flatfinder.example/pune/neighborhood-rates"
tables = pd.read_html(url, header=0, thousands=",")
print(f"Found {len(tables)} tables on the page")

df = tables[0]
print(df)
print(df.dtypes)             # numeric columns now actually numeric

# ─── If pandas can't parse it (no <table> tag) ──────────
import requests
from bs4 import BeautifulSoup

soup = BeautifulSoup(requests.get(url).text, "html.parser")
table = soup.find("table", class_="neighborhood-rates")

rows = []
for tr in table.find_all("tr")[1:]:    # skip header row
    cells = [td.text.strip() for td in tr.find_all("td")]
    rows.append(cells)

df = pd.DataFrame(rows, columns=["locality","bhk1","bhk2","bhk3"])

Always try pandas first

If your target page has data in a real HTML table, pd.read_html() will save you hours. Try it before writing a single BeautifulSoup line. If it returns garbage or fails, then drop down to manual parsing. Many financial dashboards, government statistics pages, and Wikipedia articles work with pandas straight out of the box.

4Dynamic pages · when requests returns nothing

Why some pages look full in the browser but empty to your code

Modern sites often render content with JavaScript after the page loads. The browser sees the final result; your Python requests only sees the initial blank shell. Here's what each one sees, side by side.

Two kinds of pages: server-rendered (the HTML arrives already filled in — old-school news sites, most government pages, Wikipedia) and client-rendered (the HTML arrives nearly empty and JavaScript builds it in your browser — most modern apps like Zomato, Swiggy, Instagram). The second kind needs a real browser to scrape, which means a tool like Selenium or Playwright instead of plain requests.
Same URL · what requests sees vs what your browser shows
✓ Server-rendered page requests works
What requests.get(url).text returns:
<div class="listing"> <h3>2 BHK Kothrud</h3> <span class="price">₹28,000</span> </div> <div class="listing"> <h3>1 BHK Koregaon</h3> ... <!-- All the data is right here in the HTML -->
BeautifulSoup can read it. You're done.
⚠ Client-rendered page requests fails
What requests.get(url).text returns:
<div id="root"></div> <script src="/static/app.bundle.js"></script> <!-- Listings? Nowhere. JavaScript will add them --> <!-- AFTER the page loads in a browser. --> // requests.get() never runs that JS // → you get an empty <div id="root">
You'll need a real browser to see the data.
How to tell which kind you're dealing with: in your browser, right-click → "View Page Source" (NOT "Inspect"). If you can find the listing text in the raw source, requests + BeautifulSoup will work. If the source is mostly JavaScript and the data is missing, you need Selenium or Playwright — or look for a hidden JSON API the JavaScript itself is calling (often the cleaner option). 
# For JavaScript-rendered pages: spin up a real headless Chrome
from selenium import webdriver
from selenium.webdriver.chrome.options import Options
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from selenium.webdriver.common.by import By

opts = Options()
opts.add_argument("--headless=new")        # run invisibly
driver = webdriver.Chrome(options=opts)

driver.get("https://flatfinder.example/pune")

# Wait for the listings to actually appear in the DOM
WebDriverWait(driver, 10).until(
    EC.presence_of_element_located((By.CLASS_NAME, "listing"))
)

# Now you can hand the rendered HTML to BeautifulSoup
from bs4 import BeautifulSoup
soup = BeautifulSoup(driver.page_source, "html.parser")
listings = soup.select(".listing")

driver.quit()

# ─── BETTER: check the Network tab in dev tools first.        ─
# If the JS itself calls /api/listings?city=pune and gets JSON, 
# skip Selenium entirely and just requests.get() that endpoint.

Hunt for the hidden API before reaching for Selenium

JavaScript-heavy sites usually fetch their data from an internal JSON API — and that API is right there in the browser's Network tab (F12 → Network → filter "XHR/Fetch"). Reload the page, click the requests, peek at the JSON responses. Nine times out of ten, you'll find a clean endpoint you can call directly with requests. Five seconds of network-tab sleuthing beats an hour wrestling with Selenium.

5robots.txt and ethics · scraping like a good citizen

The polite handshake every site offers — and most scrapers ignore

Every website can publish a robots.txt file telling bots which pages are open and which are off-limits. It's not enforceable code; it's an honor system — and a strong legal hint. Click each URL below to see whether FlatFinder's robots.txt would say "yes".

robots.txt lives at the root of any domain — e.g. flatfinder.example/robots.txt. It uses two main rules: User-agent (which bots this section applies to) and Allow / Disallow (which URL paths are OK). The * wildcard means "all bots". Ignoring it can get your IP banned, your account closed, and in some jurisdictions can be argued in court — be a good citizen.

Also note: Crawl-delay: 2 means wait 2 seconds between requests. Bake that into your loop.
import requests, time
from urllib.robotparser import RobotFileParser

# 1. Read and respect robots.txt
rp = RobotFileParser()
rp.set_url("https://flatfinder.example/robots.txt")
rp.read()

url = "https://flatfinder.example/pune/rentals"
if not rp.can_fetch("*", url):
    raise RuntimeError(f"robots.txt forbids scraping {url}")

# 2. Identify yourself honestly in the User-Agent
headers = {
    "User-Agent": "FlatFinder-Aggregator/1.0 (contact: ops@flatfinder.example)"
}

# 3. Pace yourself — respect Crawl-delay (or use a safe 2s default)
CRAWL_DELAY = rp.crawl_delay("*") or 2

# 4. Cache aggressively — re-running shouldn't re-hit the site
for page_url in urls_to_scrape:
    if page_url in already_cached:
        continue
    r = requests.get(page_url, headers=headers, timeout=10)
    save_to_cache(page_url, r.text)
    time.sleep(CRAWL_DELAY)     # breathe

# 5. Stop at the first sign of trouble
if r.status_code == 429:        # they're telling you to slow down
    raise RuntimeError("Site is rate-limiting us. Halt and rethink.")

The grandma test

Before you scrape, ask: "Would I be comfortable telling the site owner what I'm doing and why?" If yes — proceed politely. If no — that's your signal to stop. Personal-data pages (user profiles, account dashboards) are almost never OK to scrape, even when technically reachable. Public listing pages with a clear commercial purpose, scraped slowly, with a real User-Agent, and respecting robots.txt — that's the bright line.
